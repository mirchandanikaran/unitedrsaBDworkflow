Tender Intelligence User Guide

Complete guide for daily operations, scoring logic, and BD workflow usage.

Tender Dashboard | BD Workflow Dashboard | Keyword Planner

1) What This App Does

  • Collects tenders: pulls opportunities from configured international and national portals.
  • Normalizes and deduplicates: stores unified records in one database.
  • Auto-screens opportunities: generates score, decision (Go/Consortium/No-Go), lead assignment, and priority.
  • Tracks full BD lifecycle: from identification to handover using stages and checklists.

2) Daily Workflow (Recommended)

  1. Open either dashboard and click Ingest All Sources once.
  2. Open BD Workflow Dashboard and click Run Screening.
  3. Review Projects by Source, Overdue, and pipeline KPIs.
  4. Filter by stage/decision and open each item using Detail.
  5. Advance stage, complete checklist items, and update partners/proposals/reviews.

2.1) Keyword Planner

  • Use Keyword Planner to add domain-specific custom keywords.
  • Set keyword weight (1-20) to control scoring impact.
  • Enable/disable keywords without deleting them for temporary campaigns.
  • Run screening after changes to apply updated scoring to the pipeline.

3) Scoring Logic (Exact)

Each open tender gets a score from 0 to 100:

  • Keyword relevance: +35 base if matched, plus +4 per keyword (capped at +35 extra).
  • Focus geography match: +15.
  • Organization available: +5.
  • Budget available: +5.
  • Urgency: closing in 3 days +10, closing in 7 days +5.

Decision thresholds:

  • Go: score >= 70
  • Consortium: score 40-69
  • No-Go: score < 40

Priority: urgent (<=3 days), high (<=7 days), normal (more than 7 days or no closing date).

4) BD Pipeline Stages

The app supports these stages in order:

identified - screened - partner_development - eoi_preparation - technical_proposal - financial_proposal - quality_control - submitted - post_submission - negotiation - awarded - handover - closed

When you advance a stage, stage-specific checklist items are generated automatically.

5) SLA Timeline Rules

  • identified: 1 day
  • screened: 2 days
  • partner_development: 6 days
  • eoi_preparation: 10 days
  • technical_proposal: 28 days
  • financial_proposal: 28 days
  • quality_control: 3 days

Overdue items are highlighted on the BD dashboard.

6) How To Use Detail View

  • Click Detail on any pipeline row.
  • Review stage, decision, score, lead, source URL, and rationale.
  • Use Advance to move stage.
  • Complete checklist tasks via checkboxes.
  • Track proposals, reviews, partners, and team assignments in one place.

7) Dashboard Metrics Explained

  • Tracked: total opportunities in workflow table.
  • EOI+: opportunities in EOI/proposal onward stages.
  • Submitted: opportunities in submitted/post-submission/negotiation/awarded/handover.
  • Won: opportunities in awarded/handover.
  • Win Rate: won / submitted percentage.
  • Projects by Source: source-wise tender counts in database.

8) Quick Troubleshooting

  • If source counts are low, run Ingestion and wait for completion.
  • If no workflow rows are shown, run Run Screening.
  • If checklists/reviews are empty, advance to a stage that generates them (e.g. quality_control).
  • If pages appear stale, refresh browser once after long ingestion runs.
